Uma app que gere um hospital imaginário.

As versões da app ilustram os conceitos explicados nos slides.

  • Versão 1: app minimalista, consistindo em apenas 3 ficheiros vazios: hospital.html, hospital.css, hospital.js
  • Versão 2: definição inicial das classes Paciente, Especialidade e Consulta
  • Versão 3: definição das classes, especificando os seus campos
  • Versão 4: adição de métodos e construtores (para evitar que certos campos fiquem "undefined")
  • Versão 5: instanciação de objetos e agregação em coleções + o uso de métodos globais e "static"
  • Versão 6: evitar a "reinvenção da roda" com o uso de bibliotecas (EcmaScript APIs e Web APIs)
  • Versão 7: visualizar os objetos da app na interface gráfica (na página HTML, com CSS)
  • Versão 8: adição de formulários para possibilitar a definição de novos objetos na GUI
  • Versão 9: adição de algoritmos de procura, para encontrar certos objetos quando há critérios de procura
  • Versão 10: fazer com que a GUI adapte-se aquando de alterações no "business object"
  • Versão 11: reorganização do código fonte (dados de simulação num ficheiro separado)

F12 para abrir as ferramentas do programador + ESC para ver/esconder a consola do browser
